## 2️⃣ 프로젝트 소개

TT(Ticket & Trade) 는

공연 티케팅(예매)과 정가 기반 티켓 거래(Trade) 를 하나의 플랫폼에서 해결하는 올인원 공연 티케팅 서비스이다.

TT는 기존 티켓 시장의 불편함—불법 프리미엄 리셀, 복잡한 예매 경험, 취소표 확보 어려움, 친구와 함께 가기 힘든 구조—을 해결하기 위해 만들어졌다.

TT가 제공하는 핵심 경험은 다음 네 가지다:

- 🎟 **티케팅** : 선착순 + 추첨제 예매
- 🔄 **Trade** : 정가 기반 양도·교환 (완전 합법)
- 🔔 **알림** : 취소표 실시간 감지 & 자동 배정
- 💬 **Social** : 공연 관람자 커뮤니티 & 친구 찾기

> 예매 → 거래 → 관람까지 공연 관람 전 과정을 TT 하나로 완성한다.
>

## 3️⃣  기획 의도

## ❌ 기존 시장의 문제

- 프리미엄(웃돈) 리셀은 **한국 법상 명백히 불법**
- 공식 루트에서는 **정가 기반 양도/교환 기능 부재**
- 친구·가족 단위 사용자들이 **같은 좌석을 확보하기 어려움**
- 고령층·초보자에게 선착순 예매는 불리함
- 취소표 확보는 속도 싸움 + 시스템 미지원

→ 공연 팬·관람객 모두가 불편하고 비효율적인 경험을 겪고 있음.

---

## 💡 TT(Ticket & Trade)가 제안하는 해결책

### ✔ **1. 공정하고 합법적인 티켓 거래 구조**

- 프리미엄 리셀 금지 → **정가 거래만 가능**
- 판매자에게만 수수료 부과 → 법적으로 합법
- 본인 인증 기반 안전한 거래

### ✔ **2. 사용자층을 넓히는 예매 방식**

- 기존 선착순 예매 제공
- 일본식 *추첨제 예매* 도입 → 고령층·가족 단위 만족도 ↑
- 인원 먼저 선택 → 좌석 자동 배정 방식 지원

### ✔ 3**. 공연을 중심으로 한 새로운 소셜 경험**

- 같은 공연 관람자끼리 채팅
- 공연 커뮤니티 제공

## 4️⃣ 기술 스택

- **언어 / 프레임워크**
    - Java 21
    - Spring Boot
    - Spring Web / Spring MVC
    - Spring Security (JWT 인증/인가)
    - Spring Data JPA + Querydsl
- **DB / 캐시**
    - RDBMS: MySQL
    - 캐시·임시점유·세션: Redis
- **메시징 / 실시간**
    - Spring WebSocket + STOMP (좌석 실시간 점유 상태)
- **배치 / 스케줄링**
    - Spring Batch (또는 Spring Scheduling)
        - 미입금 자동 취소, 취소표 새벽 일괄 오픈 등
- **빌드 / 의존성 관리**
    - Gradle
- **인프라 / 배포 (백엔드 기준)**
    - Docker 컨테이너
    - 클라우드 서버 (AWS EC2) + Nginx Reverse Proxy
    - S3
- **모니터링 (후순위)**
    - Spring Actuator
    - Prometheus + Grafana (서버/애플리케이션 지표)
    - Jmeter (부하 테스트)
- **외부 API**
    - 본인 인증 (카카오)
